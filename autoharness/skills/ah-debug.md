# /ah-debug — 系统化调试

当用户输入 `/ah-debug <issue>` 时执行此技能。

## 工作流程

### 1. 复现问题

- 收集错误信息、触发条件、环境差异
- 形成最小复现步骤

### 2. 缩小范围

- 找到最可疑的模块、调用链和数据路径
- 记录已排除的假设

### 3. 定位根因

- 给出根因判断
- 指明失败位置或关键逻辑

### 4. 实施最小修复

- 只改与问题直接相关的代码
- 补齐回归验证

### 5. 输出结论

- 根因
- 修复方式
- 回归结果
- 后续风险

## 状态保存

每次调试完成后保存到 `.autoharness/workspace/debug-<date>.md`。

## 相关 Skills

- `/ah-execute` — 需要正式落修复代码时继续推进
- `/ah-verify` — 验证修复是否完整
- `/ah-ship` — 需要交付修复时再执行
